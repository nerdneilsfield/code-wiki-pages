permute_i_kernel — 输入置换核

一句话概括

这是一个基于LUT查表的硬件数据重排器，通过双缓冲乒乓机制和预计算地址映射，将线性输入流实时转换为符合素因子FFT算法要求的置换顺序。

核心职责

pfa1008_permute_i_wrapper 是输入端重排序流水线的第一个处理阶段，负责完成从DMA源到DFT-7计算核之间的数据格式转换。它的核心任务是：

  1. 接收：从DMA源读取按时间顺序排列的样本流（128-bit AXI4-Stream）
  2. 缓冲：使用双缓冲机制暂存数据
  3. 置换：根据预计算的LUT查找表重新排列样本顺序
  4. 输出：生成符合DFT-7计算要求的样本流

关键数据结构

1. 双缓冲存储 (buff)

static TT_DATA buff[2][4][4][NFFT/4];
#pragma HLS array_partition variable=buff dim=1
#pragma HLS bind_storage variable=buff type=RAM_T2P impl=bram
#pragma HLS dependence variable=buff type=intra false
维度 大小 含义
dim=1 (2) ping/pong 双缓冲切换
dim=2 (4) bank索引 支持4路并行访问
dim=3 (4) 复制索引 每样本4份副本
dim=4 (252) 地址深度 NFFT/4 = 1008/4

为什么需要4份副本？

每个周期需要读取8个样本，但BRAM只有2个端口。通过将同一样本存储在4个不同的bank中，可以实现8样本/周期的并行读取。

2. 置换查找表 (permute_lut)

static TT_ADDR permute_lut[4][NFFT] = { PERM_I_ADDR, ... };
#pragma HLS array_partition variable=permute_lut dim=1
  • PERM_I_ADDR 是预计算的1008点置换地址数组
  • 4份复制对应4个并行读取通道
  • 每个地址包含10位：addr[9:2]用于索引，offset[1:0]用于选择bank

核心函数详解

pfa1008_unpack —— 输入解包

void pfa1008_unpack( TT_STREAM sig_i[SSR_IO], TT_DATA (&permute_i)[8] )
{
  // 从stream[0]读取4个样本到偶数位置
  ( permute_i[6], permute_i[4], permute_i[2], permute_i[0] ) = sig_i[0].read();
  // 从stream[1]读取4个样本到奇数位置
  ( permute_i[7], permute_i[5], permute_i[3], permute_i[1] ) = sig_i[1].read();
}

设计要点：

  • 输入流是128-bit宽（4个32-bit样本）
  • 两个流交错提供8个样本
  • 注意索引顺序：先读的后放入低索引（大端序打包）

pfa1008_permute —— 置换核心

这是整个内核最复杂的部分，分为三个阶段：

阶段1：LUT查表计算地址

PERMUTE : for (int ss=0; ss < 8; ss++) {
    TT_ADDR rd_addr_use = permute_lut[ss>>1][rd_addr+ss];
    addr[ss].range(7,0)   = rd_addr_use.range(9,2);  // 高8位：BRAM地址
    offset[ss].range(1,0) = rd_addr_use.range(1,0);  // 低2位：bank选择
}
  • ss>>1：8个样本分成4组，每组共享一个LUT行
  • rd_addr+ss：在当前基地址上偏移
  • 地址解码：10位地址拆分为8位索引 + 2位偏移

阶段2：双缓冲写入

// 写入选中的ping buffer，同时复制到4个bank
buff[ping][0][0][wr_addrA] = permute_i[0];
buff[ping][0][1][wr_addrA] = permute_i[0];
buff[ping][0][2][wr_addrA] = permute_i[0];
buff[ping][0][3][wr_addrA] = permute_i[0];
// ... 类似写入其他样本

关键点：

  • 写入当前ping指向的buffer bank
  • 同一样本写入4个不同bank，为后续并行读取做准备
  • 写入地址wr_addrA/B线性递增

阶段3：双缓冲读取与多路选择

// 从非ping buffer读取（即上一轮写入的数据）
data0[0] = buff[!ping][0][0][addr[0]];
data0[1] = buff[!ping][0][0][addr[1]];
// ... 读取所有8个样本的4种可能

// 根据offset选择正确的数据
MUX_ALL : for (int ss=0; ss < 8; ss++) {
    switch ( offset[ss] ) {
    case 0 : permute_o[ss] = data0[ss]; break;
    case 1 : permute_o[ss] = data1[ss]; break;
    case 2 : permute_o[ss] = data2[ss]; break;
    default: permute_o[ss] = data3[ss]; break;
    }
}

乒乓切换逻辑：

bool last_wr = ( wr_addr == TT_ADDR(NFFT-8) );
bool last_rd = ( rd_addr == TT_ADDR(NFFT-8) );
ping    = ( last_wr == 1 ) ? !ping      : ping;
wr_addr = ( last_wr == 1 ) ? TT_ADDR(0) : TT_ADDR(wr_addr+8);
rd_addr = ( last_rd == 1 ) ? TT_ADDR(0) : TT_ADDR(rd_addr+8);

当写完一帧（1008点）时，切换ping指针，开始读取刚写完的buffer。

pfa1008_write_streams —— 输出格式化

void pfa1008_write_streams( TT_DATA (&permI)[8], TT_STREAM sig_o[SSR_IO] )

功能：

  • 使用FSM状态机管理输出打包
  • 引入启动延迟（startup计数器）对齐帧边界
  • 将8个样本重新打包成两个128-bit流

FSM状态转换：

switch ( fsm_state ) {
case 0:  ss0_data = ( save0[0], save1[7], save1[6], save1[5] ); break;
case 1:  ss0_data = ( save0[3], save0[2], save0[1], save0[0] ); break;
// ... 共7个状态
}

状态机的设计是为了将DFT-7所需的7点变换数据正确分布到两个输出流上。

HLS优化策略

1. 流水线约束

#pragma HLS pipeline II=1

强制综合工具实现每周期启动一次迭代的流水线。

2. 存储器接口优化

#pragma HLS bind_storage variable=buff type=RAM_T2P impl=bram
  • RAM_T2P：真双端口RAM，支持同时读写
  • impl=bram：使用Block RAM实现（而非分布式LUTRAM）

3. 依赖关系消除

#pragma HLS dependence variable=buff type=intra false

告知HLS工具：同一数组的不同元素之间没有读后写(WAR)或写后读(RAW)依赖，可以并行访问。

4. 数组分区

#pragma HLS array_partition variable=buff dim=1

将第一维（ping/pong）完全分区，创建独立的物理存储实例。

常见问题与调试

Q1: 输出数据错位一帧

症状：仿真结果显示数据整体偏移，但模式正确。

原因：启动延迟未正确处理。DMA源需要在有效数据前发送NFFT/8+1个空周期。

解决：检查dma_src是否正确实现了延迟注入（参见pfa1008_dma_src.cpp中的LATENCY处理）。

Q2: 偶发数据错误

症状：大部分时间正确，偶尔出现单个样本错误。

原因：ping-pong切换时序问题。可能是读写速率不匹配导致。

调试：

  1. 在波形中监控ping、last_wr、last_rd信号
  2. 确认下游消费者（DFT-7）不会反压导致停顿

Q3: HLS综合II不达标

症状：综合报告II > 1。

排查步骤：

  1. 检查buff的bind_storage是否正确设置为RAM_T2P
  2. 确认所有数组分区pragma已应用
  3. 查看依赖分析报告，是否有意外的内存依赖
