# HTTP API 和表格输出模块

本文档详细说明了 `http_api_tabular_output` 模块，负责将 HTTP API 返回的 JSON 数据格式化为用户友好的表格形式。

## 模块概述

该模块位于 `crates/ov_cli/src/output.rs`，承担着 CLI 应用中将 API 响应数据格式化输出到终端的核心职责。在一个命令行工具中，当用户执行各种命令（如搜索、资源管理、文件系统操作）时，API 返回的是 JSON 格式的数据，该模块的职能是将这些数据转换为人类可读的表格形式。

## 架构设计

### 核心职责

- **数据转换**：将 JSON 数据转换为表格形式输出
- **格式检测**：自动识别数据结构并选择最佳呈现方式
- **格式化输出**：处理不同的数据格式（表格、JSON）

### 数据流

```
HTTP API 响应
    ↓
output_formatting 模块
    ↓
终端输出
```

1. 用户在 CLI 执行命令
2. HttpClient 发起 API 请求
3. 获取 JSON 响应
4. output 模块进行格式化
5. 输出到终端

## 核心组件

### 输出格式

该模块提供了两种输出格式：
- **Table 格式**：将数据格式化为表格形式
- **Json 格式**：保持原始 JSON 格式

### 输出函数

- `output_success()` - 处理成功响应
- `output_error()` - 处理错误响应

## 数据处理

### 表格输出

模块会分析返回的数据结构，包括：
- 数组对象 - 渲染为表格
- 简单对象 - 渲染为键值对
- 复杂对象 - 自动选择渲染方式

### 格式化选项

- **compact**: 紧凑输出
- **pretty**: 格式化输出

## 依赖

- `serde` - 序列化
- `serde_json` - JSON 处理
- `unicode-width` - Unicode 宽度处理

## 扩展

该模块可扩展用于自定义输出格式
