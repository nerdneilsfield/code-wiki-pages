DDC 链（Digital Down-Conversion Chain）

概述

DDC 链模块实现了一个面向蜂窝通信标准（LTE/NR）的高性能数字下变频系统。该设计展示了如何在单个 AI Engine Tile 上通过精细的任务划分和内存布局，实现复杂的多级信号处理流水线。

想象一个无线电接收机的工作流程：首先接收到的是高频射频信号，需要经过一系列"筛选"和"降频"操作，最终得到低速率的基带信号。DDC 链就是完成这个任务的数字化实现——它包括半带滤波（降采样）、数字混频（频率搬移）和信道滤波（选择性增强）三个主要阶段。

关键设计决策

1. 双 Tile 架构

DDC 链被分配到两个相邻的 AIE Tile 上：

  • Tile 1 (xoff, yoff)：执行预处理阶段（解交织、HBF47、混频、HB 链）
  • Tile 2 (xoff, yoff+1)：执行信道 FIR 滤波

为什么选择这种划分？

在 ddc_subsystem.h 中可以看到显式的位置约束：

location<kernel>(rx_hbf47.hbf47d)      = tile(xoff, yoff);
location<kernel>(rx_chain_dyn_inst.chanFir) = tile(xoff, yoff+dir);

这种设计的考量包括：

  1. 计算负载均衡：信道 FIR（89-tap 或 199-tap）是计算最密集的阶段，独占一个 Tile 可以充分利用其 MAC 单元。

  2. 内存带宽优化：每个 Tile 有独立的本地内存 bank，分离部署减少了 bank 冲突。

  3. 流水线并行：两个 Tile 可以形成流水线，当一个 Tile 处理当前数据块时，另一个可以处理下一个/上一个数据块。

2. 显式内存 Bank 分配

// Tile 1 的 buffer 布局
single_buffer(rx_hbf47.hbf47d.in[0]);   
location<buffer>(rx_hbf47.hbf47d.in[0])   = bank(xoff, yoff, 3);
single_buffer(rx_hbf47.demixer.in[0]);  
location<buffer>(rx_hbf47.demixer.in[0])  = bank(xoff, yoff, 2);
single_buffer(rx_hbf47.hbf_chain.in[0]);
location<buffer>(rx_hbf47.hbf_chain.in[0])= bank(xoff, yoff, 3);
location<parameter>(rx_hbf47.hbf_chain.param[0]) = bank(xoff, yoff, 2);

设计意图：

  • single_buffer 指示编译器使用单缓冲模式，减少内存占用但增加同步复杂度
  • 不同的 buffer 被分配到不同 bank（bank 2 和 bank 3），避免并发访问冲突
  • 参数（滤波器状态）也独立存放，与数据 buffer 分离

3. 运行时比例分配

runtime<ratio>(inlv)     = 0.1;
runtime<ratio>(hbf47d)   = 0.25;
runtime<ratio>(demixer)  = 0.25;
runtime<ratio>(hbf_chain)= 0.25;
runtime<ratio>(chanFir)  = 0.8;

这些比例表示内核占用 Tile 计算资源的时间份额。总和可以超过 1.0，因为内核是通过数据流驱动的——当没有输入数据时，内核会空闲等待。

信道 FIR 获得最高的 0.8 比例，反映了其计算密集特性。

核心组件详解

1. ddc_chain_kernel - 信道 FIR 滤波器

这是 DDC 链的核心计算单元，实现了高阶 FIR 滤波（89-tap 或 199-tap）。

数据结构

class ddc_chain_kernel {
    static constexpr unsigned FIR89_COEFFICIENTS = 48;
    static constexpr unsigned FIR199_COEFFICIENTS = 104;
    
    alignas(32) cint16 (&ddcchanfir_ovlp)[OVLP_SIZE];
    alignas(32) cint16 (&dmBufA_DDC)[DDC_LTE_SAMPLES_PERCH4D];
};

关键设计点：

  • 系数数量不对称：虽然叫"89-tap"和"199-tap"，但实际存储的系数是 48 和 104。这是因为利用了 FIR 系数的对称性（线性相位滤波器），只需要存储一半系数。

  • 重叠缓冲区（overlap buffer）：FIR 滤波需要历史样本，ddcchanfir_ovlp 保存了前一数据块的尾部样本，用于与当前块进行卷积。

  • 对齐要求：alignas(32) 确保数据对齐到 256-bit 边界，这是 AIE 向量加载指令的要求。

循环缓冲区管理

ddc_kernel_stage.h 中的 buffer_internal 结构是理解 FIR 实现的关键：

struct buffer_internal {
    buffer_datatype * restrict head;  // 缓冲区起始
    buffer_datatype * restrict ptr;   // 当前位置
};

配合 cyclic_add 函数实现环形寻址：

inline void buffer256_incr_v8(buffer_internal * w, int count) {
    w->ptr = cyclic_add(w->ptr, count, w->head, 32);
}

为什么用 32 作为模数？

  • 缓冲区大小为 32 × 8 = 256 个复数样本
  • 使用 2 的幂次大小允许用位掩码代替昂贵的模运算
  • cyclic_add 内部可能使用 (ptr - head + count) & (size-1) 的优化

fir_89t_sym_buf 实现分析

inline void fir_89t_sym_buf(int position, v8cint16 * restrict pi, 
                            v4cint16 * restrict po,
                            buffer_internal * restrict buff_,
                            const v16int16 * restrict coeff_, 
                            unsigned samples)

参数解析：

  • position：当前处理位置，用于选择预计算的偏移模式
  • pi/po：输入/输出向量指针（每向量 8/4 个复数样本）
  • buff_：循环缓冲区描述符
  • coeff_：滤波器系数（16-bit 整数向量）
  • samples：要处理的样本数

核心算法：

  1. 初始化阶段：根据 position 选择预计算的偏移数组 ddc_fir89_loc

    char * restrict swift = loc_ + position;
loc_ = loc_ + (*swift);
*swift = ((*swift) + SSIZE) & 0xF;

    这段代码实现了轮转调度，确保不同的调用使用不同的初始偏移，平衡缓冲区访问模式。

  2. 三缓冲区策略：创建三个 buffer_internal 副本

    • _ovlp：读取历史样本
    • _ovlp_sym：对称读取（用于利用 FIR 系数对称性）
    • _ovlp_wr：写入更新后的历史

  3. 向量化 MAC 运算：

    acc1 = aie::sliding_mac_sym<4,4,1,1,1>(acc1, coe, 4, lbuff, 10, rbuff, 8);

    这条指令同时执行 4 个乘累加操作，利用 AIE 的 SIMD 能力。

2. 多级抽取架构

DDC 链采用级联抽取策略逐步降低采样率：

输入采样率 → HBF47 (2×抽取) → Demixer → HB Chain (2×抽取) → Channel FIR → 输出

为什么选择多级抽取？

  1. 抗混叠要求：根据奈奎斯特定理，抽取前必须充分滤波以去除高于新奈奎斯特频率的成分。单级实现需要极高阶的滤波器。

  2. 计算效率：半带滤波器有近一半系数为零，可以利用这一特性减少约 50% 的乘法运算。

  3. 灵活性：中间插入混频器允许在合适的中间频率进行频率搬移。

3. 解交织内核（ddcinlv_kernel）

connect(data_in, inlv.in[0]);
adf::dimensions(inlv.in[0]) = { HBF47D_IN_BLOCK_SIZE/4 };
adf::dimensions(inlv.out[0]) = { HBF47D_IN_BLOCK_SIZE/4 };

解交织内核负责重新排列输入数据的顺序。在无线通信系统中，来自 ADC 的数据通常以时间交织格式到达，需要转换为通道交织格式以便后续处理。

维度注解：HBF47D_IN_BLOCK_SIZE/4 表示以 128-bit（4 个 cint16）为单位的数据块大小。

内存模型与数据布局

1. 缓冲区大小计算

#define OVLP_SIZE ((DDC199_BUFF_SIZE)*DDC_NCH_NR + 
                   (DDC89_BUFF_SIZE+HBFD23_BUFF_SIZE)*DDC_NCH_LTE)*8

展开计算：

  • DDC199_BUFF_SIZE = 32
  • DDC_NCH_NR = 1
  • DDC89_BUFF_SIZE = 16
  • HBFD23_BUFF_SIZE = 3
  • DDC_NCH_LTE = 5

所以 OVLP_SIZE = (32×1 + (16+3)×5) × 8 = (32 + 95) × 8 = 1016 个 cint16

这种复杂的计算反映了系统支持多种配置（NR 和 LTE 模式）的需求。

2. 线程局部存储

#ifndef __X86SIM__
    alignas(32) static char ddc_fir89_loc[32] = {...};
#else
    alignas(32) static thread_local char ddc_fir89_loc[32] = {...};
#endif

条件编译的意义：

  • 实际 AIE 硬件：static 即可，每个 Tile 运行单一内核实例
  • x86 仿真环境：thread_local 确保多线程仿真时每个线程有独立的副本

3. 限制指针（restrict）

代码中大量使用 restrict 关键字：

void fir_89t_sym_buf(..., v8cint16 * restrict pi, ...)

这是向编译器承诺：该指针是访问其所指内存的唯一方式。这允许编译器生成更激进的优化代码（如消除冗余加载），但要求程序员确保不违反这一假设。

编译器优化提示

代码中包含大量 Chess 编译器专用 pragma：

for (unsigned l=0; l<LSIZE; ++l)
    chess_prepare_for_pipelining
    chess_loop_range(8,)
{
    // ...
}

含义：

  • chess_prepare_for_pipelining：指示编译器尝试软件流水线优化，将循环迭代重叠执行以提高吞吐量
  • chess_loop_range(8,)：提示编译器循环次数至少为 8，帮助其选择最优的展开因子

警告：这些 pragma 是编译器特定的，移植到其他平台时需要移除或替换。

设计权衡与替代方案

1. 固定系数 vs. 可重配置系数

当前实现使用编译时常量系数：

alignas(32) static constexpr int16_t coeffs_fir89[FIR89_COEFFICIENTS] = {...};

替代方案：通过 RTP（Run-Time Parameter）接口动态加载系数。

权衡：

  • 固定系数：零运行时开销，适合生产环境
  • RTP 可重配置：灵活性高，适合原型验证或多模式系统

2. 单核密集 vs. 多核分布式

当前设计将信道 FIR 集中在单个 Tile，其他预处理在另一个 Tile。

替代方案：将 FIR 进一步分解到多个 Tile，使用 SSR 架构。

权衡：

  • 单核密集：控制简单，适合中等吞吐量需求
  • 多核分布：峰值性能更高，但增加了通信开销和调度复杂度

3. 浮点 vs. 定点

代码使用 16-bit 定点数（cint16、int16）：

typedef cint16 TT_DATA;
typedef int16  TT_COEFF;

替代方案：使用 AIE 的浮点单元（如果可用）。

权衡：

  • 定点：更高的吞吐量（AIE 的定点 MAC 单元更多），更低的功耗
  • 浮点：开发更简单，动态范围更大，但需要更多的资源和功耗

调试与验证建议

1. 系数验证

滤波器系数从 MATLAB 生成并导出为 C 数组。验证时应检查：

  • 系数总和接近 1.0（保证通带增益 unity）
  • 对称性正确（对于线性相位滤波器）
  • 量化误差在可接受范围内

2. 缓冲区溢出检查

循环缓冲区的边界条件是常见错误源：

  • 确保 cyclic_add 的参数正确（特别是负数增量时）
  • 验证 position 参数在有效范围内（0-15）
  • 检查缓冲区初始化是否清零

3. 数值精度分析

定点实现需要仔细管理位宽：

  • 输入：16-bit 实部 + 16-bit 虚部
  • 系数：16-bit
  • 乘积：32-bit
  • 累加：48-bit（cacc48）
  • 输出右移：15-bit（DDC_SHIFT）

应验证整个数据路径的动态范围，确保不发生溢出或严重的量化噪声。

扩展与定制指南

修改滤波器规格

如需改变滤波器通带/阻带特性：

  1. 在 MATLAB 中重新设计滤波器（使用 firpm 或 fdatool）
  2. 导出系数到 coef_chFir89t.txt 等文件
  3. 更新 ddc_chain_kernel.h 中的系数数组
  4. 可能需要调整 DDC_SHIFT 以防止溢出或最大化 SNR

支持更多载波

当前 demixer_5c_kernel0 支持 5 个载波。要增加数量：

  1. 修改 demixer 内核实现
  2. 调整后续的 buffer 大小计算
  3. 考虑是否需要额外的 Tile 资源

移植到其他平台

将本设计移植到非 AIE 平台（如 GPU 或 DSP）需要：

  1. 替换向量指令（aie::sliding_mac_sym 等）为平台等效实现
  2. 重新设计内存布局（移除 bank 约束）
  3. 实现循环缓冲区的平台特定版本
  4. 调整数据类型和对齐要求
